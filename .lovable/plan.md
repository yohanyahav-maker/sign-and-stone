

# הבנת הזרימה והקישורים — כל הכפתורים בכל מסך

כן, אני מבין את כל הזרימה. הנה מפת הניווט המלאה לפי הסקיצה:

---

## מסך 1: Login (`/login`)

| כפתור | פעולה |
|--------|-------|
| "התחברות" | שולח אימייל+סיסמה → מעביר ל-`/projects` |
| "התחבר עם Google" | Google OAuth → מעביר ל-`/projects` |
| "התחבר עם Facebook" | Toast "בקרוב" (אין Facebook OAuth מוגדר) |
| "אין לך חשבון? הירשם" | מעביר למצב הרשמה באותו מסך |
| "שכחת סיסמה?" | מעביר ל-`/reset-password` |
| אייקוני פוטר (email/Instagram/TikTok) | פותח `mailto:` / לינק חיצוני לרשתות |

---

## מסך 2: רשימת פרויקטים (`/projects`)

| כפתור | פעולה |
|--------|-------|
| כרטיס פרויקט | מנווט ל-`/projects/:id` (דף פרויקט) |
| FAB (+) כחול | מנווט ל-`/projects/new` (יצירת פרויקט) |
| כתר (אם Pro) | אינדיקציה בלבד, ללא ניווט |
| ניווט תחתון "פרויקטים" | נשאר ב-`/projects` |
| ניווט תחתון "הגדרות" | מנווט ל-`/settings` |
| אייקוני פוטר חברתי | לינקים חיצוניים |

---

## מסך 3: דף פרויקט (`/projects/:id`) — שינוי מהותי

| כפתור | פעולה |
|--------|-------|
| חץ חזרה (→) | מנווט ל-`/projects` |
| אייקון WhatsApp (ירוק) | `https://wa.me/{client_phone}` — פותח WhatsApp עם הלקוח |
| אייקון טלפון | `tel:{client_phone}` — חייגן |
| אייקון אימייל/SMS | `mailto:{client_email}` — שולח מייל |
| "שינוי חדש" (בגריד) | מנווט ל-`/projects/:id/changes/new` |
| "העלאת מסמך" (בגריד) | פותח file picker (PDF) |
| "גלריה" (בגריד) | מציג את FileGallery של הפרויקט (inline או מודאל) |
| "צלם תמונה" (בגריד) | פותח מצלמה (`capture="environment"`) |
| "ממתין לאישור" (בגריד) | מציג רשימת שינויים מסוננת — סטטוס `sent` |
| "שינויים מאושרים" (בגריד) | מציג רשימת שינויים מסוננת — סטטוס `approved` |
| "העלאת וידאו" (בגריד) | פותח file picker לוידאו |
| 3 נקודות / עריכה | מנווט ל-`/projects/:id/edit` (פרטי לקוח) |
| ניווט תחתון | כרגיל |

---

## מסך 4: פרטי לקוח (`/projects/:id/edit`)

| כפתור | פעולה |
|--------|-------|
| חץ חזרה (→) | מנווט ל-`/projects/:id` |
| כתר (אם Pro) | אינדיקציה בלבד |
| 3 נקודות | תפריט (מחיקת פרויקט / עריכת פרטי פרויקט) |
| Toggle "פורטל לקוח" | מפעיל/מכבה `client_portal_enabled` |
| "שמור" | שומר שינויים לDB → חוזר ל-`/projects/:id` |
| ניווט תחתון | כרגיל |

---

## מסך 5: פרופיל (`/settings`)

| כפתור | פעולה |
|--------|-------|
| "שמור שינויים" | שומר full_name, company_name, business_name לDB |
| "התנתקות" | מתנתק → מעביר ל-`/login` |
| אייקוני פוטר חברתי | לינקים חיצוניים |
| ניווט תחתון | כרגיל |

---

## זרימות נוספות (קיימות, לא משתנות)

| מסך | ניווט |
|------|-------|
| `/projects/:id/changes/new` | וויזארד 3 שלבים → שומר → חוזר ל-`/projects/:id` |
| `/projects/:projectId/changes/:changeId` | פרטי שינוי + PDF + ציר זמן |
| `/projects/:projectId/changes/:changeId/send` | שליחה לאישור (Slide to Send) → חוזר לפרטי שינוי |
| `/portal/:token` | פורטל לקוח — אישור/דחייה (חיצוני, ללא BottomNav) |

---

## סיכום

```text
Login ──→ Projects ──→ ProjectDetail ──→ NewChange (wizard)
                │           │                    └→ ChangeDetail → SendChange
                │           │
                │           ├→ EditProject (פרטי לקוח)
                │           ├→ WhatsApp / Phone / Email (חיצוני)
                │           ├→ Gallery (inline)
                │           └→ Upload (file picker / camera / video)
                │
                └→ Settings (פרופיל)
```
